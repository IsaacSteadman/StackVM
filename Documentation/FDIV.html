FDIV ("Floating Point Divide" Instruction)

The FDIV instructions, number 108 (0x6C) through 111 (0x6F), all pop 2, N byte operands off the stack.

  • B
->FDIV->
  • Result

The first operand popped off the stack is B, the second popped is A.

All variants push the an N byte float onto the stack that represents the equivalent C expression: A / B

FDIV2 works on a 16 bit float (N = 2 bytes).

FDIV4 works on a 32 bit float (N = 4 bytes).

FDIV8 works on a 64 bit float (N = 8 bytes).

FDIV16 works on a 128 bit float (N = 16 bytes).

FDIV2 is assigned opcode 108 (0x6C).

FDIV4 is assigned opcode 109 (0x6D).

FDIV8 is assigned opcode 110 (0x6E).

FDIV16 is assigned opcode 111 (0x6F).
