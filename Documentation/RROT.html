RROT ("Right Bitwise Rotate" Instruction)

The RROT instructions, number 28 (0x1C) through 31 (0x1F), all pop a byte off the stack indicating how many bits to rotate by.

Stack Diagram

  • A (n bytes)
  • B (1 byte)
->RROT->
  • Result (n bytes)

All variants push the same number of bytes as were popped off, minus the byte for the shift. The bits are rotated towards the least significant bit.

RROT1 rotates an unsigned 8 bit integer (1 byte).

RROT2 rotates an unsigned 16 bit integer (2 byte).

RROT4 rotates an unsigned 32 bit integer (4 byte).

RROT8 rotates an unsigned 64 bit integer (8 byte).

RROT1 is assigned opcode 28 (0x1C).

RROT2 is assigned opcode 29 (0x1D).

RROT4 is assigned opcode 30 (0x1E).

RROT8 is assigned opcode 31 (0x1F).
