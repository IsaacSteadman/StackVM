CMP (Compare Instruction)

The CMP instructions, number 88 (0x58) through 95 (0x5F), all pop 2, N byte operands off the stack.

Stack Diagram

  • A
  • B
->CMP->
  • Result

The first operand popped off the stack is B, the second popped is A.

All variants push a 1 byte signed integer onto the stack that represents the following equivalent C expression:
A < B ? -1 : (A == B ? 0 : 1)

In other words, CMP pushes a negative number if A < B, a zero if A == B, or a positive number if A > B

CMP1(S) works on a 8 bit integer (N = 1 byte).

CMP2(S) works on a 16 bit integer (N = 2 bytes).

CMP4(S) works on a 32 bit integer (N = 4 bytes).

CMP8(S) works on a 64 bit integer (N = 8 bytes).

CMP1 is assigned opcode 88 (0x58).

CMP1S is assigned opcode 89 (0x59).

CMP2 is assigned opcode 90 (0x5A).

CMP2S is assigned opcode 91 (0x5B).

CMP4 is assigned opcode 92 (0x5C).

CMP4S is assigned opcode 93 (0x5D).

CMP8 is assigned opcode 94 (0x5E).

CMP8S is assigned opcode 95 (0x5F).
